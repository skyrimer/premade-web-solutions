Ultimate slider guide

Slider is a common way to show similar content-wise or purpose-wise information. It is widely spread among the world, and every web developer should know how to use it. However, because every user got used to the sliders, developers had to put more features into sliders to make them more accessible and better looking.

That's why I use Swiper for my sliders. Despite being quire heavy (34 kb), it can serve everything you need, want or even think of.

So, let's get started! Firstly, you have to connect external CSS styles and the Javascript file. There are a lot of way to do it that you can find here, but I will use the easiest: cdn

Now, when all the preparation is done, you can write HTML. It will have to consist of main container with class swiper. Inside it, you will have a wrapper with class swiper-wrapper. Finally, you will have all the slides with class swiper-slide and all the content insight.

An example code will look like this:


<div class="swiper">
  <!-- Additional required wrapper -->
  <div class="swiper-wrapper">
    <!-- Slides -->
    <div class="swiper-slide">
      <h2>Slide 1</h2>
    </div>
    <div class="swiper-slide">
      <h2>Slide 2</h2>
    </div>
    <div class="swiper-slide">
      <h2>Slide 3</h2>
    </div>
  </div>
  <!-- If we need pagination -->
  <div class="swiper-pagination"></div>
  
  <!-- If we need navigation buttons -->
  <div class="swiper-button-prev"></div>
  <div class="swiper-button-next"></div>
  
  <!-- If we need scrollbar -->
  <div class="swiper-scrollbar"></div>
</div>

I am also showing where to put other html elements that you might need, because I am going to refer to them later

At this moment we are done with HTML, and we just need to initialise our slider using Javascript. It is done similarly to other js libraries. We create a new object with the selector (all sliders are wrapper in container swiper class so we are going to target it) as first parameter and the dictionary of settings as the second parameter.

The activision of the simplest slider would look like this:


new Swiper(".swiper")

Slider speed

speed will define the number of milliseconds that it takes to go from one slide to another.


// Speed of the slider
speed: 400,

Navigation

In Swiper settings you have to add dictionary navigation with keys nextEl and prevEl. For each key, you pass the selector that will target your navigation arrows.


// Navigation
navigation: {
  prevEl: ".swiper-button-prev",
  nextEl: ".swiper-button-next",
},

Also, you will need to add this arrow elements in HTML. Like this:


<div class="swiper-button-prev"></div>
<div class="swiper-button-next"></div>

Pagination

In Swiper settings you have to add a dictionary pagination with several keys:

  • el to identify your pagination element

  • clickable to make the bullets clickable, so if the user clicks on it, the corresponding slide will be shown

  • dynamicBullets to make the bullets dynamic. If it is set to true, the further the bullets are from active bullet, the smaller they will be.

  • renderBullet to make your own custom bullets. It should be the function that takes 2 parameters: index of the element and classname of the bullet. It must return the string that is going to be injected as a node.

Your Javascript might look like this:


// Pagination
pagination: {
  // Activates the pagination element
  el: ".swiper-pagination",
  // Makes the bullets clickable
  clickable: true,
  // Makes the bullets dynamic
  dynamicBullets: true,
  // Create custom bullet
  renderBullet: (index, className) => {
    return `<span class="${className}">${index + 1}</span>`
  }
},

In terms of HTML, you just need to add pagination element:


<div class="swiper-pagination"></div>

However, it is not the only type of pagination you can have. There are also fraction and progressbar.

fraction shows the number of current slide relative to the total amount of slides. The settings for it would look like this:


// Pagination
pagination: {
  // Activates the pagination element
  el: ".swiper-pagination",
  // Fraction
  type: "fraction",
  // Create custom fraction
  renderFraction: (currentClass, totalClass) => {
    return `Photo <span class="${currentClass}"></span> from <span class="${totalClass}"></span>`
  }
},

progressbar shows the current slide as the progression from the total amount of slides. To activate it you just need to change the type:


// Pagination
pagination: {
// Activates the pagination element
el: ".swiper-pagination",
// Progressbar
type: "progressbar"
},

Scrollbar

It might seems like it would do exactly the same as progressbar, but there is a main difference. scrollbar fills with accent color only current position, while progressbar also fills the space of previous slides. Besides, scrollbar can be used to navigate through slides, while progressbar can not do it.

So, how to make a scrollbar? You have to add another dictionary where you specify the element using el key and draggable if you want to make navigation through the scrollbar available.


// Scrollbar
scrollbar: {
  el: ".swiper-scrollbar",
  // Makes the scroll draggable
  draggable: true,
},

Also, you have to add an element with class swiper-scrollbar


<div class="swiper-scrollbar"></div>

Mouse and keyboard control

There are 2 different dictionaries for mouse and keyboard

keyboard has enabled key that enables keyboard control (ðŸ˜²how surprisingðŸ˜²), and pageUpDown in order to control the slides with "PageUp" and "PageDown" buttons.

mousewheel has sensitivity that will determine how fast you will be able to scroll with a mouse.

All together, it will look like this:


// Keyboard controls
keyboard: {
  // Enable keyboard control
  enabled: true,
  // Allows to navigate using PageUp and PageDown
  pageUpDown: true,
},
// Mouse controls
mousewheel: {
  // Sets the sensitivity of the scrolling
  sensitivity: 2,
}

Dynamic height

It might seem useless, but it is one of the most important things in the sliders. Commonly, we can have different heights of the images or different amount of text inside the slides, which can create some weird whitespace. In order to eliminate it, you just need to set autoHeight to true.


autoHeight: true,

Slides per view

slidesPerView is property is in favour of the amount of slides displayed. It can be set both as integer or even fraction. In case of fraction, it will cut the last slide, and show the proportional part of it relative to the surd you have set.


// Amount of slides per view
slidesPerView: 4,
// slidesPerView: 4.5, - will show 4 and a half slides

Slider spacing

spaceBetween will be the spacing between the slides. It can take any positive integer or surd, and use it as the amount of pixels between the slides.


// Spacing between slides
// 40 = 40px
spaceBetween: 40,

Looped or infinite slides

loop will make the slides the effect of infinite amount of slides. In reality, he as soon as the user will go to the next slide after the last slide, he will be redirected to the first slide.


// Infinite slides
loop: true,

Autoplay

This setting will allow the slider to go to the next slide on its own. You will need only 3 properties:

delay is the amount of milliseconds before going to another slide.

stopOnLastSlide will set whether the slider will stop autoplaying on the last slide or not.

disableOnInteraction will make the autoplaying stop when the user interacts with the slider


// Autoplay
autoplay: {
  // Delay between autoplaying
  delay: 4000,
  // Stop on the last slide
  stopOnLastSlide: true,
  // Turn of as soon as the user interact with the slider
  disableOnInteraction: true,
},

Fade effect

It is the simplest swiper effect. It makes the slides, overlap each other, but using opacity, the user will see the fading animation.

To activate this effect, you need to set the corresponding name to the effect key and define fadeEffect dictionary with crossFade set to true. It will prevent the slides nesting, where the user can see several slides at the same time.


// Fade
effect: "fade",
fadeEffect: {
  // Prevents content overlap
  crossFade: true,
},

Flip effect

Similarly to fading, flip effect will make all the slider overlap, but this time the rotating effect will appear

To activate it, you need to set the corresponding name to the effect key and define flipEffect with limitRotation set to true to prevent overlapping content, and slideShadow to make the realistic drop shadow. 


// Flip
effect: "flip",
flipEffect: {
  // Creates shadow
  slideShadow: true,
  // Will show only active slide
  limitRotation: true,
},

Cube effect

Similarly to flipping, the cube effect will make the slides rotate, but will make it look like you are rotating a cube.

To activate it, you need to set the corresponding name to the effect key and define cubeEffect with shadow and slideShadows to make the cube look more realistic. Unless your design covers it, you should always turn on the shadows with this effect, otherwise the cube would look unrealistically offseted. 


// Cube
effect: "cube",
cubeEffect: {
  // Creates shadow
  shadow: true,
  slideShadows: true,
},

Cover flow effect

Honestly, I have no idea how to explain this effect. It's either my English is too bad or there are no suitable words for this. So, the best what I can do is to give you the example with the best use of such effect.

To implement it, you need to set the corresponding name to the effect key and define coverflowEffect with rotate that defines the angle of slide rotation, stretch that will define how many pixels of the adjusting slides will overlap, and optional slideShadows that will make your slides realistic.


// Cover flow
effect: "coverflow",
coverflowEffect: {
  // Angle of rotation
  rotate: 20,
  // Overlap
  stretch: 50,
  // Slide shadows
  slideShadows: true,
},

Adaptive properties

Logically, there might be some issues with slider on different screen sizes. Therefore, the library has implemented adaptiveness with breakpoints. It is a dictionary that with takes the integer (breakpoint width). For each breakpoint, you can chain the properties that you want to apply for particular screen width. Note that the all the breakpoints are set using mobile first technique. For instance, if you are setting slidesPerView: 2, for small screens, and then not define the same property on larger screen, it will take the property from the smaller screen.


// Adaptive properties
breakpoints: {
  320: {
    slidesPerView: 1,
  },
  480: {
    slidesPerView: 2,
  },
  992: {
    slidesPerView: 3,
  },
}

Image lazy loading

It is an important feature that you want to use for almost all the images in your slider.

In HTML, you will have to move the image source from src to data-src. Besides, each image that needs to be lazy loaded must have a class swiper-lazy. You might also want to place an element with classswiper-lazy-preloader after each image that you are loading lazily. Here is the example of possible HTML structure


<div class="swiper">
  <div class="swiper-wrapper">
    <div class="swiper-slide">
      <img data-src="/images/image2.png" alt="Image" class="swiper-lazy">
      <div class="swiper-lazy-preloader"></div>
    </div>
    <div class="swiper-slide">
      <img data-src="/images/image2.png" alt="Image" class="swiper-lazy">
      <div class="swiper-lazy-preloader"></div>
    </div>
    <div class="swiper-slide">
      <img data-src="/images/image3.png" alt="Image" class="swiper-lazy">
      <div class="swiper-lazy-preloader"></div>
    </div>
    <div class="swiper-slide">
      <img data-src="/images/image4.png" alt="Image" class="swiper-lazy">
      <div class="swiper-lazy-preloader"></div>
    </div>
    <div class="swiper-slide">
      <img data-src="/images/image5.png" alt="Image" class="swiper-lazy">
      <div class="swiper-lazy-preloader"></div>
    </div>
  </div>
</div>

In JavaScript, you will need to disable image preloading by setting preloadImages to false, and activate lazy loading with corresponding setting. To make lazy loading almost invisible you want to set loadOnTransitionStart and loadPrevNext to true. It will make the images to load on transition and also load previous slide's image and next slide's images.


// Lazy
preloadImages: false,
lazy: {
  loadOnTransitionStart: true,
  loadPrevNext: true,
},
// If you have multiple slidesPerView set
watchSlidesProgress: true,

And you are done! You have saved a tone of traffic for your users.

Parallax effect

Seems like I am finishing endless number of Swiper features. Anyway, parallax. In html, there aren't any elements that you have to add. Everything you should do is to add some data attributes.

  • data-swiper-parallax will define by how much will the content be displaced. It can be both integer (pixel displacement) and string with percent (proportion that is going to be displaced by). Both values can be positive (the content moves less) or negative (the content moves more).

  • data-swiper-parallax-opacity will define from which opacity will content start the animation.

  • data-swiper-parallax-scale will define from which transform: scale() will content start the animation.

  • data-swiper-parallax-duration will set the transition for the effects.
