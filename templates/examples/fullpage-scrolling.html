Fullpage scrolling with an amazing Javascript Library

I find fullpage scrolling absolutely amazing. It seems like the website is guiding you through all the content, showing and telling you the story. Actually, it is quite easy and implement this feature into your website.

For this tutorial, I am going to use library called Fullpage js. There is only one limitation of this library: it is allowed to use it for open source projects, however, to access all the extensions and templates, you will have to pay. You can read about it in detail on their pricing page here.

So, how to implement this feature? This will require specific HTML structure and a bit of Javascript.

In HTML, you need to have a container with sections inside (I refer sections as classes, but semantically you will most likely use sections with class of section). In the examples below, I have a main container with id="fullpage" with sections inside. The library also supports vertical scrolling: you just need to use slider class inside the section.

Finally, we come to the Javascript part. On our side, it works like any other library. We need to specify new object and as the first parameter we pass a selector that will get our main container (in the examples above it is #fullpage). As the second parameter we can pass the dictionary with all wanted configurations. This is how Javascript looks in my example:


const navigationSectionNames = ['Slide 1', 'Slide 2', 'Slide 3', 'Slide 4', 'Slide 5',
                                'Slide 6', 'Slide 7', 'Slide 8', 'Slide 9', 'Slide 10']
const scroller = new fullpage("#fullpage", {
  autoScrolling: true,
  scrollingSpeed: 900,
  // Navigation
  navigation: true,
  navigationTooltips: navigationSectionNames,
  showActiveTooltip: true,
  // If you use slides
  controlArrows: false,
  slidesNavigation: true,
  // Anchoring
  // anchors: ['list', 'of', 'your', 'anchors', 'here'],
  // For animations
  onLeave: (origin, destination, direction) => {
    toAnimate = destination.item.querySelector(".animate")
    if (toAnimate) {
      toAnimate.classList = "content_wrapper animate going"
    }
    fromAnimation = origin.item.querySelector(".animate")
    if (fromAnimation) {
      fromAnimation.classList = "content_wrapper animate"
    }
  },
})

In the majority of cases, this is all the configuration you will need. Here is the explanation for it:

  • autoScrolling is the property that activates fullpage scrolling.

  • scrollingSpeed is an integer that determines the time taken (in milliseconds) to go from one section or slide to another. The default is 700, and as for me, it is a bit too quick, so I like to slightly increase it.

  • navigation is a boolean that activates the small navigation menu. By default, it is just a circle set on the middle right of the browser window. By clicking on those, the user can skip thorough sections and go directly to the destination. However, without any additional set-up, this navigation is not extremely useful.

  • navigationTooltips is a list of strings for each section. Now, it will display the corresponding label for active slide or if the user hovers over the bullet point, it will also display the label. It makes the navigation, way more user-friendly and simple to navigate through.

  • showActiveTooltip is a boolean that will make the current tooltip stand out by applying special class (by default: it will display the label and make the bullet point bigger). Because it adds special class, it can be easily customisable using CSS.

  • controlArrows is a boolean that will either show (true) or hide (false) control arrows that helps the user to navigate through slides.

  • slidesNavigation is a boolean that will toggle the navigation menu, but for slides.

  • anchors is a list of strings that will add ids to every section, when the user goes to it. It might be usefull, if you have a lot of redirects to different sections, however, it makes "go back" and "go forward" keys go thought the slides, that you have visited, which might make it a bit annoying.

  • onLeave is a function that is going to be triggered as soon as the user goes to another section. This is the easiest way to have different onscroll animations, because the function must take 3 parameters: origin (section or slide which is leaving), destination (section or slide which is going to be shown) and direction (down or up)
