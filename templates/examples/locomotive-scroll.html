A way to make smooth scrolling

You might have spotted that on some websites scrolling feels weird. It seems to have some sort of inertia. For instance this website

This technique is called üöÇlocomotive scrollingüöÇ because it smoothly starts and smooth finishes to scroll as if you are on the Swiss train. It does not drastically affect the user experience but improves it. The simplest way to realise this effect is to use Locomotive-Scroll JavaScript Library. It is easy to set up, and it has a bunch of features to satisfy all your needs including parallax.

To start with, you need to connect both JavaScript (for animations) and Css (for custom scrollbar). You can use both npm or cloud package


npm install locomotive-scroll

<!-- or -->

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/locomotive-scroll@4.1.3/dist/locomotive-scroll.min.css">
<script src="https://cdn.jsdelivr.net/npm/locomotive-scroll@4.1.3/dist/locomotive-scroll.min.js"></script>

After you have connected all necessary stuff, you have to activate locomotive scrolling using JS: 


const scroll = new LocomotiveScroll({
  el: document.querySelector("[data-scroll-container]"),
  smooth: true,
});

In your HTML you have to add at least 1 data-scroll-container attribute to any element.


<div class="continer" data-scroll-container></div>

That's itüëç! Refresh your document, and you will see the magic of a better scrolling experience. The values set by default work perfectly and look good for the majority of cases. However, if you want to customise something, You just need to pass some new parameters to the dictionary of arguments. There are many of them, but I will list the most useful in my opinion.

  • direction - specifies the direction of the effect. Can be vertical and horizontal. It is useful in case you want to make animated elements moving horizontally.

  • lerp - smoothness of the scrolling. Take values from 0 to 1, where 1 is no effect and 0 is no scrolling at all. Just a values to play around.

  • multiplier - factor applied to the scrolling. The higher the value, the more page scrolls.

  • firefoxMultiplier - factor applied to the scrolling for FireFox browser.

  • touchMultiplier - factor applied to the scrolling on touchscreen devices. Generally, you want this to be higher than the default value, because on phones you have to scroll way more than on any big screen.

  • Adding data-scroll-section to the elements inside data-scroll-container might improve performance.

  • General note. If you have some JavaScript code that changes the hight of the content (for instance: syntax highlighting that adds paddings), you need to run this piece of code first, and then initialise locomotive scroll.

  • There is only 1 issue with implementing this feature: you can not scroll using middle button, and there is nothing to do with itüò•. On one hand, it is actually good, because the users won't scroll through without even noticing the content. On the other hand, people who are used to the middle button will suffer a bit from it.

For more information, use the official documentation, which describes all the powers of the library in detail.
