Page transition

One of my favourite effects on websites is page transition. It makes the process of going from one webpage to another smoothüçÉ by adding fancy animation. For instance, this website also contains fading page transition!. If you click on any link, all the content will fade out, and the new one will fade in when it is ready, while the navigation stays the same.

There are several ways to achieve this affect: with native JavaScript, Barba Js, Highway Js, but we are going to use Swup Library because of its simplicity

Firstly, you need to have the same markdown for all the pages that you want to add the animation. It should have a container with an id="swup" and class=""transition-fade. The id defines the content that is going to be reloaded (the content inside will undergo the animation) and the class will be used in CSS to animate everything.


<main id="swup" class="transition-fade">
  <!-- Your contenet here that is going to be refreshed -->
</main>
<script src="/path-to-file/swup.js"></script>

In terms of JavaScript, everything you need to do, is to activate the library with just one line


const swup = new Swup();

Finaly, you have to add your styling for animation using css. .transition-fade will be added to your content, when the content will be reloaded. In our case, we need to add opacity for fading effect and transition to make transition smooth


.transition-fade {
  transition: 0.4s;
  opacity: 1;
}

html.is-animating .transition-fade {
  opacity: 0;
}

And that's it! You have introduced a fancy page transitionüòé.

However, I have to note that you will also need a few features. The main issue of library is that it does not manually reload CSS and JavaScript, which means that you have to do it yourself. For this purpose, there are 2 custom event listeners: contentReplaced that is going to be activated after the content was replaced with new information, and willReplaceContent that is called right before the content is reloaded
