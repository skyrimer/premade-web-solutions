Interactive charts

Charts are primarily used in the admin panel of the website. Therefore, the utilities to construct these charts will go with the site's system control management. However, it is also expected that some small charts appear now and then to represent some information (user timeline or difference between using your product and other software). So, how are we supposed to do it? Hopefully, Chart Js managed to provide a library that can solve all our problems.

In Html, you need to create a canvas element with some sort of identification that we can access through JavaScript. If you wish to have fixed values for height and width, you can declare them here. Like this:


  <canvas class="chart"></canvas>

In CSS, there is nothing particularly related to the tutorial. Just make sure that your chart has enough space.

In JavaScript happens, all the magic. After connecting the library, you need to query for your element and access its 2d context. Then pass it to the Chart object as the first parameter, followed by the chart config. That would be it, if not a million available settings. Hopefully, I can summarise them here, and if you still need additional functionality, you can access the official documentation:


new Chart(canvas.getContext("2d"), chartConfig)

Config is a dictionary that consists of 3 parameters: type, data and options. type refers to the graph's representation (line, bar, bubble, scatter, pie, polar area or doughnut). data is an object that includes your data and its customisation. options are other graph customisation that does not directly refer to data.

Data must include one dataset: your actual data. In the dataset, you can also have customisation: the label for this data, background colour, border colour etc. The above data must be labelled with any string using the label key.

Options have layout control like padding or animation and additional information like the legend, axis, title, subtitle etc. There are a lot of settings you can add here, but the most important I have mentioned above.

I think that this information is sufficient enough for you to get started with the library. I know that I didn't cover everything you might need in your life as I did with sliders, but assuming that the library is so extensive and there are so many small settings that you will use once per project, I doubt that they have to be explained in detail. In addition, the documentation and examples will be better and more up-to-date. If you want to see all primary types of charts, you are welcome to check out the pen.
