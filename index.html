How to set a custom cursor?

Making custom cursor with image

If there is an image or an SVG prepared, it is simple to do it with a single CSS property. 


body {
    cursor: url("path/to/your/image"), auto;
}

The first argument is the URL with a path to the image. The second argument is the fallback in case the image can not be displayed. "Auto" in this case is the default behaviour of the cursor (as if there is no property set), however, you can set any value you want.

The list of all the values here

Making custom with CSS

Another way to make a custom cursor on the website is to "draw" your own on the website.

You have to remove default cursor using css


* {
    cursor: none;
}

Then you add the HTML that you will customize as a cursor. It can be any element, but it would be more logical to use either div or span because they are used for styling and id because there has to be only one cursor on the webpage.


<div id="cursor"></div>
<div id="aura"></div>

Here comes your time to shine and design! You can make almost whatever you want, but you have to always think of user experience because your custom cursor should be both pretty and useful. As an example, I will make a common doubled circle. The example is below:

Because you are removing the cursor, it will trigger all the events (clicks or hovers), but it will not change the state itself. The only way to solve it is to manually add all the cursor changes using CSS andJavaScript.

Here in CSS, I am both defining default (when the user is just exploring the website), hidden (when the mouse is out of the website dimensions) and active (when the user is hovering curtain objects) states.

position: absolute and z-indexis used to display this element over the others (z-index should be higher than any other element has).

user-select: none and pointer-events: none are set to none to make all the mouse events trigger the area under the middle dot. Otherwise, the user would always select the dot and the following circle.

#cursor is the small middle dot that instantly follows the cursor. There is a tiny lag between the mouse movement and the dot on the screen, but there is nothing to do with it because it is a browser restriction.

#aura is the big circle following the dot with a larger delay to create the parallax effect. The same circle can be drowned as a pseudo-element, however, it would follow the dot without any delay, which has to be taken into account.

Everything else is just styling that can be changed however you want. The example provided is just a sample that looks nice and can be taken to adapt to your own needs.


/* Cursor styling */
#cursor,
#aura {
  position: absolute;
  z-index: 10;
  border-radius: 50%;
  top: -5rem;
  left: -50vw;
  transform: translate(-50%, -50%);
  user-select: none;
  pointer-events: none;
}
#cursor {
  width: var(--size-cursor);
  height: var(--size-cursor);
  background: var(--color-accent);
  transition: opacity 0.1s ease-out,
              transform 0.1s ease-out;
}
#aura {
  width: var(--size-aura);
  height: var(--size-aura);
  border: 1px solid var(--color-accent);
  background-color: rgba(248, 0, 174, 0.1);
  transition: all 0.1s ease-out;
}
/* When cursor is out of the website */
#cursor.hidden,
#aura.hidden {
  opacity: 0.05;
  transform: scale(0.005);
}
/* When the cursor hovers the link */
#cursor.active {
  opacity: 0;
}
#aura.active {
  width: calc(var(--size-aura) + 1rem);
  height: calc(var(--size-aura) + 1rem);
  border-color: var(--color-active);
}

Using JavaScript, we are attempting to make a custom cursor 'alive'. This is why you have to manipulate all the states yourself.

For each change of state of the cursor, there is a separated event listener (mouse of the page, hovers the link, does not hover the link...)

To move the dot and circle, we are adding the event listener on the document, which will adjust the position when the mouse is moved.

Interesting fact, there are 2 ways to move the circle: using transform: translate(e.pageX, e.pageY) and top: pageY px; left: pageX px. The first way is working faster and smoother on old machines, but might cause some problems with centering (the code uses transform: translate(-50%, -50%) with position: absolute to center everything). The second solution cause way less problems to the programmer, but might cause performance issues on old machines.


// Declaring the constants
const cursor = document.getElementById("cursor");
const aura = document.getElementById("aura");
const links = document.getElementsByTagName("a");
// As soon as mouse moves the it will move the custom cursor to its new positions
document.addEventListener("mousemove", (e) => {
  const pageOffset = `top: ${e.pageY}px; left: ${e.pageX}px`
  cursor.setAttribute("style", pageOffset)
  aura.setAttribute("style", pageOffset)
});
// Adds new class when the cursor is out of the webpage
document.addEventListener("mouseout", (e) => {
  cursor.classList.add("hidden");
  aura.classList.add("hidden");
});
// Removes the class when the cursor is on of the webpage
document.addEventListener("mouseover", (e) => {
  cursor.classList.remove("hidden");
  aura.classList.remove("hidden");
});

for (let i = 0; i < links.length; i++) {
  const link=links[i];
  // Adds special class when user hovers over a link
  link.addEventListener("mouseover", (e)=> {
    cursor.classList.add("active");
    aura.classList.add("active");
  });
  // Removes special class when user does not hover over a link
  link.addEventListener("mouseout", (e) => {
    cursor.classList.remove("active");
    aura.classList.remove("active");
  });
}

That's it! Despite taking some time and energy custom cursor makes your website a bit fancier than everyone else's. Great job👍

A way to make smooth scrolling

You might have spotted that on some websites scrolling feels weird. It seems to have some sort of inertia. For instance this website

This technique is called 🚂locomotive scrolling🚂 because it smoothly starts and smooth finishes to scroll as if you are on the Swiss train. It does not drastically affect the user experience but improves it. The simplest way to realise this effect is to use Locomotive-Scroll JavaScript Library. It is easy to set up, and it has a bunch of features to satisfy all your needs including parallax.

To start with, you need to connect both JavaScript (for animations) and Css (for custom scrollbar). You can use both npm or cloud package


npm install locomotive-scroll

<!-- or -->

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/locomotive-scroll@4.1.3/dist/locomotive-scroll.min.css">
<script src="https://cdn.jsdelivr.net/npm/locomotive-scroll@4.1.3/dist/locomotive-scroll.min.js"></script>

After you have connected all necessary stuff, you have to activate locomotive scrolling using JS: 


const scroll = new LocomotiveScroll({
  el: document.querySelector("[data-scroll-container]"),
  smooth: true,
});

In your HTML you have to add at least 1 data-scroll-container attribute to any element.


<div class="continer" data-scroll-container></div>

That's it👍! Refresh your document, and you will see the magic of a better scrolling experience. The values set by default work perfectly and look good for the majority of cases. However, if you want to customise something, You just need to pass some new parameters to the dictionary of arguments. There are many of them, but I will list the most useful in my opinion.

  • direction - specifies the direction of the effect. Can be vertical and horizontal. It is useful in case you want to make animated elements moving horizontally.

  • lerp - smoothness of the scrolling. Take values from 0 to 1, where 1 is no effect and 0 is no scrolling at all. Just a values to play around.

  • multiplier - factor applied to the scrolling. The higher the value, the more page scrolls.

  • firefoxMultiplier - factor applied to the scrolling for FireFox browser.

  • touchMultiplier - factor applied to the scrolling on touchscreen devices. Generally, you want this to be higher than the default value, because on phones you have to scroll way more than on any big screen.

  • Adding data-scroll-section to the elements inside data-scroll-container might improve performance.

  • General note. If you have some JavaScript code that changes the hight of the content (for instance: syntax highlighting that adds paddings), you need to run this piece of code first, and then initialise locomotive scroll.

  • There is only 1 issue with implementing this feature: you can not scroll using middle button, and there is nothing to do with it😥. On one hand, it is actually good, because the users won't scroll through without even noticing the content. On the other hand, people who are used to the middle button will suffer a bit from it.

For more information, use the official documentation, which describes all the powers of the library in detail.

Floating labels for inputs

One of the best thing you can do while creating forms is to label every single input with its label. It make the website more intuitive and user-friendly. One of the ways to add some fancy micro-interactions for the website is to make the labels float.

Floating label is a term that is used to demonstrate that the label moves as soon as user focuses the input. You can see it on the example below:

In terms of HTML, structure maters a lot. You have to have a wrapper with an input followed by a label. It is important to have a label afterwards because we are using pure CSS. For this example, you will also need a placeholder for the input. There are other ways to do it using :focus and :valid, but we find this one a bit simpler. 


<div class="input-wrapper">
  <input type="text" id="input" placeholder="Put any text here">
  <label for="input">Click</label>
</div>

For CSS, you can style the input and label however you want, however, there are some things to think of. You have to set position: relative to the wrapper, so you can position the label. Another thing to be watched is padding: you have to make sure that there is enough space both for the input text and for the label text. Here, we are lowering the input text by setting greater padding-top than padding-bottom so there is enough space for label text.

Finally, we are using the magic selector input:placeholder-shown + label to change styling for the label. This selector will cover all the labels after input when the placeholder value is not visible. In particular, we are moving the label higher using transform: translate(), but you can also use top: and left: 


input {
  font-size: 1.2rem;
  width: min(100%, 50rem);
  padding: 2em 2rem 0.5em;
  color: var(--color-text);
  background-color: var(--color-background);
  border-bottom: 1px solid var(--color-background);
  border-radius: 0.5em;
}

/* So the placeholder does not overlap with the label */
input::placeholder {
  color: transparent;
}

input:placeholder-shown+label {
  transform: translateY(-50%);
  font-size: 1rem;
}

label {
  font-size: 0.9rem;
  transform: translate(0, calc(-50% - 1em));
  user-select: none;
  position: absolute;
  top: 50%;
  left: 2rem;
  color: var(--color-accent);
  transition: all 0.3s ease-in-out;
}

And that's it👍! You can use the same structure for any input, select or textarea. If you wish to have a look at it once again, here is a document for you.

Reveal navbar on scroll

Another animation that does not change the content but slightly improves user experience is animated navbar. Fixing this element is a common trick to provide users with the persistent excess to the navigation. On the other hand, it takes up some space, and might be unnecessary element during website exploration.

The effect that I am going to show will hide the navbar when the user is scrolling down, and reveal it when the user is scrolling up.

Firstly, you will need a page with a lot of content, so this effect makes sense. Possible HTML might look like this:


<nav class="scroll-reveal">
  <img src="/logo.svg" alt="">
  <ul>
    <li><a href="#">Thirst link</a></li>
    <li><a href="#">Second link</a></li>
    <li><a href="#">Third link</a></li>
  </ul>
</nav>
<div class="content">
  <section>
    <h2>Thirst section</h2>
  </section>
  <section>
    <h2>Second section</h2>
  </section>
  <section>
    <h2>Third section</h2>
  </section>
  <section>
    <h2>Fourth section</h2>
  </section>
  <section>
    <h2>Fifth section</h2>
  </section>
</div>

Then in css, you have to fix navigation and declare additional styling that will be manipulated using JavaScript.

You can use both position: fixed or position: sticky, but the second variant might cause some problems with background (because it pushes all the content, when the navbar is hiding at the top of the page, it will reveal the background of the space behind it)

nav:focus-within must be declared to show the navbar if the user selects the elements using the keyboard.

And nav.is-scrolling that is going to be used for hidden state of navigation. Your CSS should look like this:


nav {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  display: flex;
  flex-wrap: wrap;
  justify-content: space-between;
  padding: 1.5rem 4rem;
  background-color: #021520;
  transition: transform 500ms ease-in-out;
}

nav:focus-within {
  transform: translateY(0);
}

nav.is-scrolling {
  transform: translateY(-100%);
}

JavaScript contain the most complicated part.

You have to initialise all variables: navbar element, current position of the page, and throttle variable that is going to be used later. There are different ways to get, but if you don't want to support ancient versions of Internet Explorer, construction from the example will do its job. 


const nav = document.querySelector("nav");
let previousScrollPosition = window.scrollY || window.pageYOffset;
let throttleTimer;

Then you have to declare 3 functions: for getting whether the user scrolls or not, scroll handler, and the interval function that is going to be passed as an event listener.

Whether the user scrolls or not is easy to get. You have to have a previous position and compare it with current position. If current position is greater than previous position, the user has scrolled down. In the other case, the user either hasn't scroll or scrolled up.

The second function just toggles the navbar state.

The last function is quite interesting. You could have just passed the previous function to an event listener, however, it will be called, each time the user scrolls, which will cause performance issues. One of the ways to fix it is to call the function every curtain amount of time when the user scrolls. This is how throttle works. A better explanation you can see here. The code will look like this: 


const isScrollingDown = () => {
  let scrolledPosition = window.scrollY || window.pageYOffset
  let isScrollDown = scrolledPosition > previousScrollPosition ? true : false
  
  previousScrollPosition = scrolledPosition;
  return isScrollDown;
};

const handleNavScroll = () => {
  if (isScrollingDown() && !nav.contains(document.activeElement)) {
    nav.classList.add("is-scrolling")
    return
  } 
  nav.classList.remove("is-scrolling")
};

const throttle = (callback, time) => {
if (throttleTimer) return;

throttleTimer = true;
setTimeout(() => {
callback(); throttleTimer = false
}, time);
};

Now, when all the preparation is done, you just need to add scroll event listener to the document. The second parameter in the throttle function is the interval in milliseconds.


window.addEventListener("scroll", () => {
  throttle(handleNavScroll, 200);
})

You are done👍! Congratulations on finishing this effect.
